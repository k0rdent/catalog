    Traffic Tracing with Tempo#

    Modern platform environments rarely consist of a single Kubernetes cluster. Organizations commonly operate multiple clusters across cloud providers, edge environments, and on-premise infrastructure. Understanding how requests flow through distributed systems becomes increasingly difficult in such environments. When a user request travels through multiple services and clusters, identifying latency bottlenecks or failures without distributed tracing is challenging.

    The k0rdent Catalog simplifies deploying and managing production-ready platform services across multiple Kubernetes clusters. Catalog applications are packaged as service templates that can be installed once and then deployed consistently across many clusters using MultiClusterService resources.

    This demo demonstrates how the k0rdent Catalog can be used to deploy a centralized distributed tracing stack consisting of:

    • Istio – provides a service mesh that automatically captures distributed traces between services
    • OpenTelemetry Collector – receives and processes tracing data generated by the mesh
    • Tempo – stores and indexes distributed trace data
    • Grafana – visualizes traces and allows exploring request flows across services
    • kube-prometheus-stack – used as the example application whose internal traffic will be traced

    Together, these components provide a multi-cluster distributed tracing solution:

    • Istio automatically instruments service-to-service communication and generates tracing data
    • OpenTelemetry Collector gathers and forwards traces from the clusters
    • Tempo provides scalable distributed trace storage
    • Grafana enables visualization and analysis of request flows and latency across services
    • kube-prometheus-stack serves as the example workload whose traffic is traced through the mesh

    Using the k0rdent Catalog service templates, this tracing stack can be installed once and then deployed automatically to any number of clusters. This ensures consistent configuration while reducing operational complexity.

    In this guide, we will deploy the distributed tracing stack using k0rdent Catalog templates and demonstrate how service-to-service traffic within the kube-prometheus-stack application can be traced and analyzed through Grafana and Tempo.

    Prerequisites#

    Deploy k0rdent v0.2.0: QuickStart

    Install template to k0rdent#

    helm upgrade --install traefik oci://ghcr.io/k0rdent/catalog/charts/kgst --set "chart=traefik:39.0.5" -n kcm-system
helm upgrade --install tempo oci://ghcr.io/k0rdent/catalog/charts/kgst --set "chart=tempo:1.24.4" -n kcm-system
helm upgrade --install opentelemetry-collector oci://ghcr.io/k0rdent/catalog/charts/kgst --set "chart=opentelemetry-collector:0.145.0" -n kcm-system
helm upgrade --install istio-base oci://ghcr.io/k0rdent/catalog/charts/kgst --set "chart=istio-base:1.28.3" -n kcm-system
helm upgrade --install istiod oci://ghcr.io/k0rdent/catalog/charts/kgst --set "chart=istiod:1.28.3" -n kcm-system
helm upgrade --install raw oci://ghcr.io/k0rdent/catalog/charts/kgst --set "chart=raw:2.0.2" -n kcm-system
helm upgrade --install kube-prometheus-stack oci://ghcr.io/k0rdent/catalog/charts/kgst --set "chart=kube-prometheus-stack:81.6.3" -n kcm-system

    Verify service template#

# kcm-system   istio-base-1-28-3               true
# kcm-system   istiod-1-28-3                   true
# kcm-system   raw-2-0-2                       true

    Deploy stack#

    apiVersion: k0rdent.mirantis.com/v1beta1
kind: MultiClusterService
metadata:
  name: tempo
spec:
  clusterSelector:
    matchLabels:
      group: demo
  serviceSpec:
    services:
    - template: traefik-39-0-5
      name: traefik
      namespace: traefik
      values: |
        traefik:
          deployment:
            kind: DaemonSet
          ports:
            web:
              port: 8000
              hostPort: 80
            websecure:
              port: 8443
              hostPort: 443
    - template: tempo-1-24-4
      name: tempo
      namespace: tracing
      values: |
        tempo:
          tempoConfig:
            distributor:
              receivers:
                otlp:
                  protocols:
                    grpc:
                    http:
            storage:
              trace:
                backend: local
                local:
                  path: /var/tempo
            metrics_generator:
              registry:
                external_labels:
                  source: tempo
          persistence:
            enabled: true
            size: 2Gi
    - template: opentelemetry-collector-0-145-0
      name: opentelemetry-collector
      namespace: tracing
      values: |
        opentelemetry-collector:
          image:
            repository: "otel/opentelemetry-collector-k8s"
          mode: deployment
          config:
            receivers:
              otlp:
                protocols:
                  grpc:
                    endpoint: 0.0.0.0:4317 # Ensure it listens on all interfaces
            exporters:
              otlp:
                endpoint: "tempo.tracing.svc.cluster.local:4317"
                tls:
                  insecure: true
              debug:
                verbosity: basic # basic, normal, detailed
            service:
              telemetry:
                metrics:
                  address: "0.0.0.0:8888"
                logs:
                  level: "info"
              pipelines:
                traces:
                  receivers: [otlp]
                  exporters: [otlp, debug]
          ports:
            otlp:
              enabled: true
              containerPort: 4317
              servicePort: 4317
              hostPort: 4317
              protocol: TCP
            metrics:
              enabled: true
              containerPort: 8888
              servicePort: 8888
              protocol: TCP
    - template: istio-base-1-28-3
      name: istio-base
      namespace: istio-system
    - template: istiod-1-28-3
      name: istiod
      namespace: istio-system
      values: |
        istiod:
          meshConfig:
            enableTracing: true
            defaultProviders:
              metrics:
              - prometheus
              tracing:
              - otel
            defaultConfig:
              proxyLogLevel: debug
              tracing:
                provider:
                  name: otel
                sampling: 100
            extensionProviders:
              - name: otel
                opentelemetry:
                  service: otel-opentelemetry-collector.tracing.svc.cluster.local
                  port: 4317
    - template: raw-2-0-2
      name: raw
      namespace: raw
      values: |
        raw:
          resources:
          - apiVersion: v1
            kind: Namespace
            metadata:
              name: prometheus
              labels:
                istio-injection: enabled

          - apiVersion: telemetry.istio.io/v1alpha1
            kind: Telemetry
            metadata:
              name: mesh-tracing
              namespace: istio-system
            spec:
              tracing:
                - providers:
                    - name: otel
                  randomSamplingPercentage: 100
    - template: kube-prometheus-stack-81-6-3
      name: prometheus
      namespace: prometheus
      values: |
        kube-prometheus-stack:
          prometheus:
            prometheusSpec:
              maximumStartupDurationSeconds: 300
              storageSpec:
                volumeClaimTemplate:
                  spec:
                    accessModes:
                      - ReadWriteOnce
                    resources:
                      requests:
                        storage: 5Gi
            ingress:
              enabled: true
              hosts: ['prometheus.example.com']
          grafana:
            sidecar:
              datasources:
                httpMethod: GET
            ingress:
              enabled: true
              hosts: ['']
            additionalDataSources:
              - name: Tempo
                type: tempo
                access: proxy
                url: http://tempo.tracing:3200
                jsonData:
                  httpMethod: GET
                  serviceMap:
                    datasourceUid: prometheus

    Testing in child cluster#

    Get Grafana admin user password: 

    kubectl get secret prometheus-grafana -n prometheus -o jsonpath="{.data.admin-password}" | base64 -d; echo

    Access Grafana UI and use admin username and password to log in. Use Grafana "Explore" view to inspect onbtained tracing data.
