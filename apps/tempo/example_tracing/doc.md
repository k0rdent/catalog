# Traffic Tracing with Tempo
Modern platform environments rarely consist of a single Kubernetes cluster. Organizations commonly operate multiple clusters across cloud providers, edge environments, and on-premise infrastructure. Understanding how requests flow through distributed systems becomes increasingly difficult in such environments. When a user request travels through multiple services and clusters, identifying latency bottlenecks or failures without distributed tracing is challenging.

The k0rdent Catalog simplifies deploying and managing production-ready platform services across multiple Kubernetes clusters. Catalog applications are packaged as service templates that can be installed once and then deployed consistently across many clusters using MultiClusterService resources.

![](./assets/tracing.png){ align="right", width="600" }

This demo demonstrates how the k0rdent Catalog can be used to deploy a centralized distributed tracing stack consisting of:

- Istio – provides a service mesh that automatically captures distributed traces between services
- OpenTelemetry Collector – receives and processes tracing data generated by the mesh
- Tempo – stores and indexes distributed trace data
- Grafana – visualizes traces and allows exploring request flows across services
- kube-prometheus-stack – used as the example application whose internal traffic will be traced

Together, these components provide a multi-cluster distributed tracing solution:

- Istio automatically instruments service-to-service communication and generates tracing data
- OpenTelemetry Collector gathers and forwards traces from the clusters
- Tempo provides scalable distributed trace storage
- Grafana enables visualization and analysis of request flows and latency across services
- kube-prometheus-stack serves as the example workload whose traffic is traced through the mesh

Using the k0rdent Catalog service templates, this tracing stack can be installed once and then deployed automatically to any number of clusters. This ensures consistent configuration while reducing operational complexity.

In this guide, we will deploy the distributed tracing stack using k0rdent Catalog templates and demonstrate how service-to-service traffic within the kube-prometheus-stack application can be traced and analyzed through Grafana and Tempo.
